## 1. Описание прибора

Прибор управления и контроля с обратной связью с участием **ПИ-регулятора**
(пропорционально-интегрального) для управления воздушным клапаном
(дроссель-клапаном, шибером) и/или скоростью (оборотами) вентилятора
посредством подачи сигнала напряжением **0–10 В**.

**Входной сигнал** — меняющееся напряжение от **термоанемометрического
датчика** скорости воздуха/паров (датчик с горячей нитью, охлаждаемой
проходящим потоком).

**Органы управления и индикации**:
- ЖК-дисплей **2 × 16 символов** (белый на синем), цифробуквенный,
  с задней подсветкой.
- Клавиши (с индикаторными светодиодами):
  - Включение **вентиляции** (с LED)
  - Включение **освещения** вытяжного шкафа (с LED)
  - **Mute** — отключение звукового сигнала тревоги
  - **Авто/Ручной** (Auto/Mano) — переключение режима управления
  - **Выбор/Активация разъёмов** (RB) — включение розеток
  - **Аварийная кнопка E** — аварийный режим
  - Многофункциональные клавиши

**Конфигурация** — через отдельные меню, защищённые паролем:
- Калибровка датчика: нуль, усиление, диапазон выхода, параметры ПИД.
- Пороги минимальной/пониженной скорости, задержка аварии.
- Техническое обслуживание (счётчик моточасов + порог).
- Регистратор данных (RS-485).

**Конструкция** — два модуля (связаны кабелем до 2,5 м):
1. **Лицевая панель** — клавиатура + ЖК 2×16.
2. **Блок питания и коммутации** — бокс на DIN-рейке.

## 3. Электрические соединения

### Клеммы и разъёмы (Блок питания и коммутации)

**J1** — внешний датчик температуры:
```
GND, Ext.Temp, +5V
```

**J3 (RJ-45)** — к клавиатуре управления:
```
N.C., Coil FAN, VOut, COIL LAMP,
N.C., Coil HORN, N.C., Coil SOCKET
```

**J4 (RJ-11)** — датчик скорости:
```
+5V, +5V, GND, IN-, IN+, GND
```

**MT1** — последовательный канал RS-485:
```
GND, RS485 N, RS485 P
```

**MT2** — дискретные входы:
```
IN Отказ инвертера, GND,
IN Стекло вверх,    GND
```

**LK1** — перемычка «5 V Прямой» (link закрыт).
**LK2** — перемычка батареи («on» = link закрыт).
**J5** — разъём для перепрограммирования прибора.

Провода клеммника: 0,2–2,5 мм² (AWG 24-12). Кабель панель↔БПК —
8-жильный, RJ-45, длина 2,5 м.

## 4. Режимы работы

### Стандартная комплектация
- Режим ожидания
- Программирование
- Нормальный / сокращённый (ночной) режим работы
- Режим **предупреждения** — скорость ниже порога короче задержки
- Режим **тревоги** — скорость ниже порога дольше задержки
- Конфигурация параметров:
  регулировка датчика — ПИ-параметры — пороги/уставки — задержка тревоги
- Эксплуатация
- Регистрирующее устройство (опционально)

### С буферной батареей
- **Активный режим аварийного отключения**:
  переход из Программирования/Конфигурации через **1 с** после пропадания
  питания; мигает «Black Out», зуммер прерывисто, все реле открыты,
  выход 0 В; реагирует только на **Mute/PRG** (выключает зуммер).
- **Пассивный режим аварийного отключения**:
  из режима ожидания через 1 с после пропадания питания; прибор
  уходит в low-power, команды игнорирует.

По восстановлении питания прибор возвращается в режим ожидания;
активация — вручную. LK2 должна быть замкнута; если прибор простаивает
> 1 мес., LK2 открыть, чтобы не разрядить батарею.

## 5. ФУНКЦИОНАЛЬНОСТЬ КНОПОК

### 5.1 **Mute / PRG**
- Короткое: **отключает зуммер** тревоги.
- Короткое: **вход/выход из режима программирования параметров**.
- В конфигурации короткое: **переход к следующему параметру** в подменю.

### 5.2 **Вкл/Выкл (ON/OFF)**
- В ожидании / программировании короткое: переключает режим работы.
- **Удержание > 3 с** → **пониженная скорость (Ночной режим)**.
  На этом этапе короткое нажатие → возврат в ожидание.
- В конфигурации короткое: **уменьшает значение** выбранного параметра.

### 5.3 **Лампа**
- В программировании короткое: **переключает реле LAMP**
  (светодиод над кнопкой = состояние реле).
- В конфигурации короткое: **увеличивает значение** параметра.
- В режиме **ожидания — НЕАКТИВНА**.

### 5.4 **Авто/Ручной (Auto/Mano)**
- Короткое: **переключает** между регулируемым и постоянным выходом
  (состояние — надписью на дисплее).
- В конфигурации короткое: **переход к следующему окну настроек**.

### 5.5 **Выбор / Активация разъёмов (RB)**
- Короткое: активирует реле **SOCKET** (время зависит от настроек).
- В конфигурации короткое: **переход к предыдущему окну настроек**.

### 5.6 **Аварийная кнопка E**
- Короткое: переводит прибор в **аварийный режим** — **выходы на
  максимум**, на дисплее «EMERGENZA».
- Повторное короткое: возврат в предшествовавшее состояние.

## 6. Включение и работа прибора

1. Подача питания → **диагностика** («POWER ON» на дисплее).
2. Автоматический переход в **автоматическую вентиляцию**
   с нормальной скоростью до заданных значений.
3. В режиме вентиляции нажатие ON/OFF → все LED гаснут, все реле
   размыкаются → **Режим ожидания**.
4. Повторное ON/OFF → переход к настройкам. Порядок:
   - Через **0,5 с** замыкается реле вентилятора, загорается LED.
   - На **1 с** выдаётся **максимальная выходная мощность**,
     на дисплее — «ВЕНТИЛЯЦИЯ».
   - По истечении **4 с** — регулируемый вывод по настройкам,
     на дисплее «ВЕНТИЛЯЦИЯ».

Прибор работает:
- в **автоматическом режиме** — ПИ-регулятор по анемометру, до уставки;
- в **ручном режиме** — прямое присвоение % выхода.

Переключение — кнопкой **A/M**. Состояние автоматический/ручной при
выключении **не сохраняется** в ПЗУ.

При падении скорости ниже порога → последовательность тревог
(см. §8). После установленного числа моточасов (ORE) на каждом
включении ~10 с выводится предупреждение о **техобслуживании**,
пока оно не сброшено в защищённом паролем меню.

Опция: **RS-485** — чтение/запись параметров, запись данных скорости
каждые *n* секунд с меткой времени; сессия защищена паролем.

## 7. Экраны дисплея

> Ширина поля = 16 символов. Ниже «·» = значащий пробел.

### POWER-ON
3 с все LED активны, зуммер 1 с:
```
POWER  ON       
Ver.1.5          
```

### Автоматическая регулировка (AUTO)
```
Sp·0.50··v·0.50m/s
AUTO·········27°C
```

### Ручная регулировка (MANO)
```
Sp···50%··v·0.50m/s
MANO·········27°C
```

### Пониженная скорость (AUTO R — символ «r» = reduced)
```
Sp·0.50··v·0.50m/s
AUTOR········27°C
```

### STAND-BY
```
··········12:24:33
Standby··08/04/11
```

### Аварийный режим (кнопка E)
```
Sp·0.50··v·0.50m/s
EMERGENZA····27°C
```

### Активная розетка (socket)
На экранах AUTO / EMERGENZA появляется специальный символ в свободной
зоне слева-вверху или справа-внизу — обозначает активное реле socket.

### Тревога — «LOW SPEED»
Каждые 10 с на 2 с включается зуммер; сообщения перебираются по
очереди: **LOW SPEED**, **INVERTER FAULT**, **GLASS OPEN**.
```
Alarm           
Low·Speed        
```

## 8. Управление тревожными сообщениями

- При возникновении условия каждые **10 с** на **2 с** звучит зуммер,
  на дисплее — «ALARM».
- Если активно несколько тревог — выводятся по очереди.
- Источники:
  - **LOW SPEED** — скорость ниже минимального порога.
  - **INVERTER FAULT** — ошибка входа инвертера (MT2).
  - **GLASS OPEN** — открыто верхнее стекло (MT2).
- Нажатие **Mute** — зуммер замолкает, включится снова при следующем
  возникновении тревоги. При устранении причины — сообщения и зуммер
  пропадают сами.
- **Важно**: любые тревоги **игнорируются первые 45 с** включённой
  вентиляции.

## 9. Дата/Время

Вход в меню установки — **одновременное удержание `PRG` + `↓` 3 с**.
Выход с сохранением — **удержание `PRG` 3 с**.

- Поля день/месяц/год мигают — меняются `↑`/`↓`.
- `A` → следующее поле, `B` → предыдущее.
- `PRG` → переход к окну регулировки Времени (и обратно).
- При корректировке минут секунды обнуляются.

```
Set···Date·08/04/11
Set···Hour·12:24:33
```

## 10. Программирование параметров

Вход — **удержание `PRG` > 3 с**. Выход — повторное удержание > 3 с.
**Таймаут неактивности — 2 мин.** Индикатор режима — специальный
символ в правой верхней зоне.

Доступ защищён разными паролями по каждой «зоне» (PASS1…PASS4).
Пароли выдаёт продавец.

В окне ввода пароля:
- мигающее поле изменяется `↑` / `↓` (0…9);
- `A` — следующее поле, `B` — предыдущее;
- `PRG` — подтвердить; при ошибке — выход из программирования.

```
PASSWORD:··P····
················
·········****····
```

### PASS1 — рабочие уставки пользователя

| Окно                                   | Значение/ед.   | Дисплей (нижняя строка)|
|----------------------------------------|----------------|------------------------|
| **Setpoint** — уставка потока          | м/с            | `Setpoint P 0.50m/s`   |
| **Setpoint R** — уставка пониж. скор.  | м/с            | `Setpoint R P 0.50m/s` |
| **Alarm Low** — порог тревоги          | м/с            | `Alarm Low P 0.50m/s`  |
| **Alarm Low R** — порог в «ночи»       | м/с            | `Alarm Low R P 0.50m/s`|
| **Alarm Time** — задержка тревоги      | с (0…180)      | `Alarm Time P 003s`    |
| **Manual Set** — NOR/MEM               | строка         | `Manual Set MEM`       |

Примечания:
- Alarm Low — когда скорость «нормальная», нажатие **R (B)** в окне
  включает/выключает автоотключение розетки при падении скорости ниже
  порога; состояние индицируется символом внизу слева.
- Alarm Time = 0 → срабатывание немедленное.
- Manual Set:
  - `NOR` — при переходе AUTO→MANUAL сохраняется последнее
    автоматически рассчитанное значение %.
  - `MEM` — при переходе AUTO→MANUAL возвращается последнее
    значение, ранее выставленное вручную.

### PASS2 — калибровка и ПИД

Только для квалифицированного персонала с анемометром и вольтметром.

| Окно                                 | Назначение                                               |
|--------------------------------------|----------------------------------------------------------|
| **ZERO** — нуль датчика              | в стоячем воздухе; ↓ до 0 В выхода; сохранить — **E**    |
| **SPAN** — диапазон датчика          | совпадение показания и анемометра; **A+B** = полная шкала|
| **OUT Zr / OUT Sp** — 0 / FS выхода  | **A** = FS, **B** = 0; `E+B` = нуль, `E+A` = FS          |
| **PID — TI / sBand**                 | `TI` сек [0–9,9]; `sBand` см/с [1–999]                   |
| **BLACKOUT** — резервное питание     | ON / OFF                                                 |

Индикативные значения на скриншотах: `TI 2.7`, `sBAND 027 cm/s`,
`ZERO OK 0.00 m/s`, `SPAN OK 0.42 m/s`, `OUT Sp OK 100 %`.

### PASS3 — техобслуживание

| Окно              | Описание                                            |
|-------------------|-----------------------------------------------------|
| **MANUTENZIONE**  | сброс счётчика моточасов (`PRESET`); нажатие `↑`    |
| **COUNT MAX**     | порог предупреждения о ТО, часы (до 9999h)          |

### PASS4 — регистратор данных

| Окно              | Описание                                               |
|-------------------|--------------------------------------------------------|
| **DATA LOGGER**   | период записи (сек), ON/OFF (`A + ↑`), очистка (`B + ↑ + ↓`) → CLEAN 3 с |

Экспорт — специальной программой, поставляемой продавцом.

## 12. Сопоставление с прошивкой

> Связь документа со сквозной реализацией в этом репозитории.

### 12.1 Кнопки ↔ прошивка

Порядок строго по §5.1..§5.6.  `BtnIndex_t` нумерует кнопки в этом же
порядке, коды `BTN_EV_*` короткого нажатия равны индексу+1.

| §   | Документ      | Пин MCU (LQFP64) | `BtnIndex_t`       | Короткое (код)    | Длинное                   |
|-----|---------------|------------------|--------------------|-------------------|---------------------------|
| 5.1 | Mute / PRG    | PB2              | `BTN_IDX_PRG=0`    | `BTN_EV_PRG=0x01` | `BTN_EV_PRG_LONG=0x81`    |
| 5.2 | Вкл/Выкл      | PB3 (JTDO)       | `BTN_IDX_ONOFF=1`  | `BTN_EV_ONOFF=0x02`| `BTN_EV_ONOFF_LONG=0x82` (≥3 с) |
| 5.3 | Лампа         | PB5              | `BTN_IDX_LAMP=2`   | `BTN_EV_LAMP=0x03`| `BTN_EV_LAMP_LONG=0x83`   |
| 5.4 | Авто/Ручной   | PB4 (JTRST)      | `BTN_IDX_AUTO_MAN=3`| `BTN_EV_AUTO_MAN=0x04` | —                    |
| 5.5 | RB (Выбор)    | PB15             | `BTN_IDX_RB=4`     | `BTN_EV_RB=0x05`  | —                         |
| 5.6 | E (авария)    | PD2              | `BTN_IDX_E=5`      | `BTN_EV_E=0x06`   | `BTN_EV_E_LONG=0x86` (резерв) |

PB3/PB4 требуют `SWJ_SWDPENABLE_REMAP` (JTAG-DP off, SW-DP on) —
сделано в `btn_init()`. Default `BTN_LONG_MS = 3000` (3 с для ночного).

### 12.2 Режимы `mode_sr` ↔ документ

| Бит                      | Док.                        |
|--------------------------|-----------------------------|
| `MB_BIT_MODE_POWER`      | Любой не-ожидающий режим    |
| `MB_BIT_MODE_WORK`       | Активная вентиляция         |
| `MB_BIT_MODE_ALARM`      | Low Speed / Inv. / Glass    |
| `MB_BIT_MODE_MANUAL`     | MANO / AUTO                 |
| `MB_BIT_MODE_STANDBY`    | Stand-by                    |
| `MB_BIT_MODE_REPEAT`     | AUTO R (пониженная)         |
| `MB_BIT_MODE_NIGHT`      | Ночной режим (hold ON/OFF)  |
| `MB_BIT_MODE_EMERGENCY`  | EMERGENZA (кнопка E)        |

### 12.3 Реле ↔ регистр `OUTPUT_STATE`

| Бит           | Клемма (J3) | Документ               |
|---------------|-------------|------------------------|
| `MB_OUT_LAMP` | `COIL LAMP` | Освещение (авто-выкл через 30 мин при выключенной вентиляции) |
| `MB_OUT_SOCKET` | `Coil SOCKET` | Розетки (замыкание ≥ 0,5 с после нажатия) |

Прочие реле (`Coil FAN`, `Coil HORN`) управляются через `MODE_SR`
(`WORK` → FAN) и флаги тревог (`ALARM_FLAGS` → HORN).

#### `BUZZER_STATE` (reg 121, отдельный регистр)

Статус зуммера тревоги (read-only):
- `0x00` — выключен;
- `0x01` — включён.

Прошивка ведёт цикл 10 с / 2 с (`BUZZER_PERIOD_TICKS = 1000`,
`BUZZER_ON_TICKS = 200`) пока активна тревога и пользователь не нажал
Mute. Сброс mute — на новой 0→1 транзиции `MB_BIT_MODE_ALARM`.

Удалённый mute через Modbus: запись `MB_CMD_BUZZER_MUTE = 0x0C` в
`MODE_CR`. Локальный mute — короткое нажатие `PRG`/`Mute` на MAIN
во время аварии или короткое нажатие `PRG` на экране ALARM.

Аппаратная интеграция: подключить пьезоэлемент через NPN/MOSFET к
свободному GPIO; в драйвере периферии читать `BUZZER_STATE` каждые
≤ 10 мс и зеркалить на пин.
